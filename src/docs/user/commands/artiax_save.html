Command: artiax save

Usage:
save filename path [format type] partlist/geomodel model-spec volume model-spec | (voldim dimensions voxelsize size) [prefix string] [tomonumber integer]

The save command is based on ChimeraX’s save command. For further information regarding the underlying structure, check the chimeraX save command documentation. The argument keys partlist or geomodel allow you to save a particle list or geomodel to a specified file. The model, format, and file path can be defined.

property meaning expected value type
filename Specifies the name (and optionally path) where the file will be saved string
format Specifies the file format (e.g., motl, relion). To see all available formats run 'open formats' in command line. string
partlist or geomodel Specifies the particle list or geomodel (e.g., #1.2.1) model-spec

Examples:

save filename model_output format motl
save filename /path/to/output format relion

Saving a particle list in RELION 5 format requires additional arguments. To compute the coordinates, the dimensions and pixel size of the corresponding tomogram are needed. This information can either be extracted from a tomogram volume already loaded in ArtiaX or specified manually. To correctly populate the 'rlnTomoName' column in the created star file, you can specify the desired prefix that precedes the tomogram number. When saving a newly created particle list, a tomogram number can be specified, which will be written in 'rlnTomoName'.

property meaning expected value type
volume Specifies the volume from which tomogram dimensions and pixelsize will be extracted (e.g., #1.1.1) model-spec
voldim Specifies volume dimensions (e.g., 500,500,1000). Alternativly, specify volume List of 3 floats
voxelsize Specifies volume voxelsize (e.g., 8.73). Alternativly, specify volume float
prefix Specifies prefix before tomogram number in 'rlnTomoName' (e.g., TS_, tomo). string
tomonumber Optional. Specifies corresponding tomogram number. Input will overwrite tomogram number in 'rlnTomoName'. (e.g., 42). If no number is specfied, existing numbers will be written out integer

Examples:

save filename /path/to/output format relion5 partlist #1.2.1 voldim 896,696,250 voxelsize 11.52 prefix tomo
save filename /path/to/output format relion5 partlist #1.3.1 volume #1.1.1 prefix tomo
save filename /path/to/output format relion5 partlist #1.2.1 volume #1.2.1 prefix tomo tomonumber 42
