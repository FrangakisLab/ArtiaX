General: ArtiaX

Artiax supports several common cryoET particle list file formats. Files can be opened and saved using the default ChimeraX open and save commands with some additional parameters.

Usage:
open filename format format-name [csvpath filename | csvsuffix string]

Usage:
save filename partlist model-spec [csvpath filename | csvsuffix string]

Examples:

    open example.em format motl
    open example.star format star

    save example.mod partlist #1.2.3 csvsuffix iter1
    save example.tbl partlist #1.2.1

File Types

format format-name suffix description
Particle lists
Artiatomi/TOM/AV3 motivelists motl .em Binary file in EM format containing matrix of size 20xN. Positions in file[7:10, :], shifts in file[10:13, :], euler angles in file[16:19, :].
IMOD model2point output coords .coords Three column (x, y, z), space delimited text file. One particle per line. No header line.
Dynamo table dynamo .tbl See dynamo documentation. Accepts any table with more than 26 columns.
TSV particle lists generic .tsv Tab-delimited generic input format with one line per particle and unlimited number of columns.

9 Expected Columns:
pos_x, pos_y, pos_z, shift_x, shift_y, shift_z, phi, the, psi

A header line must be present. Any additional named columns will be imported as metadata values.
PEET motivelist model/csv peet .mod IMOD model and optionally associated CSV-based motivelist. Use csvpath or csvsuffix for specifying the optional CSV input or output
Relion star files star .star
RELION 2, 3 and 4 STAR files. For multi-section star files, the first loop containing rlnCoordinateZ will be read. If rlnTomoName is present, names are expected to contain 1 underscore, and follow the RELION 4 tutorial guidelines.

rlnCoordinateX/Y/Z must be present. ArtiaX first looks for rlnOriginX/Y/Z and then for the rlnOriginX/Y/ZAngst equivalent if not found. Angles (rlnAngleRot/Tilt/Psi) are optional and read if present.
